# Target Page Judgment

Use this reference when deciding whether the current page is already the real target page.

## Core distinction

Separate:
- **discovery surface**: search results, listings, recommendation feeds, landing pages, intermediate pages
- **real target page**: the actual article, post, account page, repo page, issue page, video page, or other page the user truly wants

## Rule

Do not finalize the answer while still on a discovery surface if the user task requires the real target page.

## Signs you are still on a discovery surface
- multiple candidate cards are visible
- you have not opened the actual article/post/repo/account/video page yet
- the page mostly shows search or recommendation results
- page identity is still ambiguous
- the visible text is only a snippet or preview

## Signs you are on the real target page
- the page has a stable title or identity tied to one concrete target
- the visible content belongs to one article/post/repo/video/account
- the user’s target can now be verified from page-level evidence
- the page is good enough to summarize or verify directly

## Reporting rule

If you are still on a discovery surface, say so explicitly.
Do not write as though the target page itself has already been confirmed.
